**Tooling note (affects Assessment B for all units):** the installed `impeccable` CLI uses `impeccable detect [--json] <file|dir|url>` and has **no `impeccable live` overlay server** (`/live` is an AI-harness slash command, not a CLI). JSON output is written to **stderr**, exit code 2 = findings found. Template files scanned standalone return `[]` (jsdom can't resolve the Jinja `{{ url_for }}` stylesheet link ‚Üí no computed styles). So Assessment B runs the detector against the **live URL** (`detect --json http://127.0.0.1:5001/<route>`, Puppeteer renders real CSS but **anonymous** ‚Äî login-required routes redirect, fall back to a logged-in screenshot), plus a logged-in `[Human]` tab screenshot in place of the nonexistent overlay.

**Authenticated detector harness (added Session E ‚Äî gives Assessment B a real signal on admin/login-gated routes).** `impeccable detect` has no cookie/header/auth flag (only `--fast`, `--json`), so its anonymous Puppeteer hits the `@admin_required`/`@login_required` redirect (it measures `index.html`, not the gated template ‚Äî the "no admin signal" limitation hit by Units 3, 5, 7, 10). Workaround that **does** give the detector authenticated markup: render the gated template through Flask's **test client with an admin session injected** (`sess['_user_id']='1'`; `WTF_CSRF_ENABLED=False` on the *in-process* client only ‚Äî the live dev server is untouched), write the rendered HTML (which links the real `/static/css/style.css`) to a repo-root file, serve the repo root (`python -m http.server 8765`), and run `detect --json http://localhost:8765/<file>.html`. Puppeteer then renders the **real gated DOM + real style.css** (no auth needed ‚Äî it's now a static file). Reproducible via `.critique_shots/harness.py` (ephemeral; rebuild from the snippet here). Non-mutating: only GET + the non-saving `load_field` POST. Used for Units 8 & 9; **retro-applicable to Units 3/5/7/10** at consolidation. The brand **green navbar gradient** still trips two false positives in this mode ‚Äî `ai-color-palette` "Cyan gradient" AND `dark-glow` "#00432e on dark" (both = `style.css:95` `--green-900`‚Üí`-800`).

## Global issues (recur across pages ‚Üí fix-campaign Phase 1)

These trace to `base.html` / design tokens / `style.css` and therefore recur on every page. Units 2‚Äì10 should reference these by id rather than re-reporting.

- **[G1] Low-contrast global color tokens fail WCAG AA.** `--text-muted: #8b95a2` computes **2.7‚Äì3.0:1** on the cream/white/mint backgrounds; gold accents `--gold-500: #b8993e` compute **2.7:1** on white. Used pervasively (nav subtext, footer, every `.text-muted`, gold links/`.text-gold`, points chip ~3.48:1, admin badge ~4.16:1). Detector flagged 12 instances on `/`, 20 on `/tournament/23`, **1 each on `errors/404.html` and `errors/500.html` (Unit 6) ‚Äî the single `lead text-muted` recovery sentence at 2.86‚Äì2.9:1**, the most important line on an error page rendered nearly invisible (fix: use Slate `#4a5568` ‚âà7:1 for the lead, reserve Stone for tertiary), **and 3√ó on `/register` (Unit 7) ‚Äî every form helper hint (`<small class="text-muted">`: "This is how you'll log in", "How your name appears‚Ä¶", "At least 6 characters") at 3.0:1, i.e. a first-timer's field guidance rendered sub-AA on the entry screen.** _Source: `style.css:21` (`--gold-500`), `:33` (`--text-muted`), alert at `:481-486`. Fix at the token level ‚Üí propagates. P1._
- **[G2] Role-locked status-blue leaks outside tournament status.** Two faces of the same violation: (a) the keyboard `:focus` ring shows `rgba(13,110,253,0.25)` Bootstrap blue glow on nav/links/buttons ‚Äî low-contrast on pine; (b) **confirmed on admin/payments (Unit 10): the "Paid" payment toggle is a Bootstrap-blue `rgb(13,110,253)` switch** ‚Äî blue means "active tournament" everywhere else, so a settled-payment control glows the same hue as a live event. ### Not in scope / never "fix"

- **Detector false positive ‚Äî the brand green navbar/footer gradient**, reported as both `ai-color-palette` "Cyan gradient" and `dark-glow` "#00432e on dark" (both = `style.css:95` `--green-900`‚Üí`-800`; footer at `:670`). Genuinely intentional Augusta-pine brand gradient; do **not** change the design to "fix" it. The two stops (`#00432e`/`#005c3f`) compute to hue **~161¬∞**, just past the coarse green‚Üícyan bucket boundary (~150¬∞), so the classifier mislabels the teal-leaning brand green as cyan, and `#00432e`'s ~13% lightness trips the separate `dark-glow` heuristic ‚Äî a tool-classifier limitation, not a design error. Brand anchor Pinehurst Pine `#006747` **stays** (decision 2026-05-27). The CLI has no ignore/config mechanism (only `--fast`/`--json`), so the proper fix lives at the tooling layer: the campaign **Setup step** adds a **signature-tight** known-FP filter to `.critique_shots/harness.py` post-processing ‚Äî keyed to the `#00432e`/`#005c3f` tokens + the navbar/footer selectors, **never** a loose "contains cyan" string match (which would mask a genuinely-wrong cyan gradient introduced later) ‚Äî so future re-critiques stay deterministic and noise-free without touching one pixel of correct design.
